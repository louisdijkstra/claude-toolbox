Write a commit message for the staged changes (`git diff --cached`).

Use conventional commit format (feat/fix/docs/refactor/chore/test). Subject line under 50 chars. Add a body only if the change needs explanation.

**Language Style:**
- Use natural, simple, and informal language
- Write like a human teammate, not AI
- Avoid overly formal or corporate tone
- Keep it conversational and direct
- Example: "fix login redirect" not "Implement corrective measures for authentication flow redirection"
